LLM 到底是什么？从原理到 Prompt 工程入门

不用懂数学，也能理解大语言模型的工作方式——以及怎么跟它高效对话

Friday, April 24, 2026

一句话理解 LLM

大语言模型（Large Language Model，LLM）本质上是一个超级高级的"猜词机"。你给它一句话，它猜下一个词应该是什么；把这个词接回去，再猜下一个，以此类推，直到生成完整的回答。

这个"猜"不是瞎猜，而是基于海量文本训练出的概率分布。模型见过互联网上几乎所有的公开文字——维基百科、书籍、论文、代码、论坛帖子——它学会了什么样的词跟在什么样的上下文后面最合理。

三个核心概念

1. Token：模型的"原子"

LLM 不是按"字"或"词"来处理文本的，而是按 Token。Token 可以是半个字、一个字，也可以是多个字的组合。比如：

  • "Hello" = 1 Token
  • "中国" ≈ 2 Tokens
  • "Transformer" = 1 Token（常见英文词会被整体编码）

为什么这很重要？因为模型的计费、上下文长度限制，都是按 Token 算的。一篇 2000 字的中文文章，大约等于 3000–4000 Token。

2. 上下文窗口：模型的"工作记忆"

上下文窗口（Context Window）是模型能同时"看到"的 Token 数量。早期的 GPT-3 只有 4K，现在主流模型（Claude、GPT-4、DeepSeek）已经支持 128K 甚至 200K。

这意味着你可以一次性塞进去一本短篇小说、一段完整的代码库、或者几十轮对话历史。但注意：窗口越大，模型越容易"走神"——后面的内容可能会遗忘前面的关键信息。

3. Temperature：创造力的旋钮

Temperature（温度）控制模型输出的随机性：

  • 温度 = 0：最保守，每次几乎输出相同的答案，适合代码生成、数学计算
  • 温度 = 0.7：平衡，既有逻辑又有一定的创造性，适合大多数对话场景
  • 温度 = 1.0+：更发散，可能给出意想不到的回答，适合头脑风暴、写作

Prompt 工程入门

Prompt 工程不是"玄学"，而是如何清晰地向模型表达你的需求。以下是几个经过验证的技巧：

1. 角色设定（Role Prompting）

给模型一个明确的角色，它的回答风格会立刻改变。

你是一位经验丰富的 Python 后端工程师，擅长 Django 和 FastAPI。
请帮我 review 下面这段代码，重点关注性能瓶颈和安全漏洞。

2. 清晰的指令（Clear Instructions）

模糊的请求得到模糊的结果。把"写得好一点"改成具体的标准：

❌ 帮我写个产品说明
✅ 帮我写一个 200 字的产品说明，目标用户是 25–35 岁的程序员，
   突出"无需配置即可使用"和"支持私有化部署"两个卖点，语气专业但不呆板。

3. 提供示例（Few-shot Learning）

模型擅长模仿。给几个输入-输出对，它比长篇大论的解释学得更快。

把下面的口语化描述改写成正式的 Git commit message。

输入：改了登录接口，修复了手机号验证的 bug
输出：fix(auth): 修复登录接口手机号验证逻辑

输入：加了个新功能，用户现在可以导出 Excel 了
输出：feat(export): 新增用户数据 Excel 导出功能

输入：重构了订单模块的代码，抽了个公共方法出来
输出：

4. 思维链（Chain of Thought）

对于复杂推理任务，让模型"一步步想"，准确率会显著提升。

请解决下面的问题，并在给出最终答案前，先展示你的推理过程。

一个水池有 A、B 两个进水管，A 管单独注满需要 6 小时，
B 管单独注满需要 4 小时。同时打开 A、B 两管，注满需要几小时？

5. 结构化输出（Structured Output）

如果你需要把模型的输出接入程序，明确要求 JSON、Markdown 表格等格式，并给出字段说明。

请分析以下用户评论的情感倾向，并以 JSON 格式输出：
{
  "sentiment": "positive|neutral|negative",
  "confidence": 0.0-1.0,
  "keywords": ["关键词1", "关键词2"],
  "summary": "一句话总结"
}

常见误区

  • 误区一：Prompt 越长越好 —— 冗长的提示反而会让模型迷失重点，关键信息要前置
  • 误区二：一次问完所有问题 —— 复杂任务拆成多轮对话，每轮聚焦一个子问题，效果通常更好
  • 误区三：模型什么都知道 —— LLM 的知识有截止日期，对最新事件、私有数据、实时信息无能为力，需要配合 RAG（检索增强生成）
  • 误区四：同样的 Prompt 适用于所有模型 —— Claude 喜欢详细说明，GPT-4 对结构化提示响应更好，国产模型对中文指令优化更到位

写在最后

LLM 不是魔法，它是一个概率引擎；Prompt 工程也不是黑盒艺术，而是清晰沟通的技术。

你不需要理解 Transformer 架构中的自注意力机制数学公式，也能用好这些工具。关键是：知道它能做什么、不能做什么，以及怎么把你的需求翻译成一个它"听得懂"的指令。

如果你刚开始接触 AI 开发，建议从实际场景出发——先写一个你日常工作/生活中真的需要的 Prompt，然后迭代优化它。实践比任何理论都有效。
