国内 AI Coding Plan 横评：DeepSeek、智谱 GLM、通义千问、MiniMax、Kimi

作为一个 OpenClaw 用户，模型选择直接决定了两个核心体验：响应速度 和钱包厚度

Sunday, March 29, 2026

前言：为什么要对比？

作为一个 OpenClaw 用户，模型选择直接决定了两个核心体验：响应速度 和钱包厚度 。

OpenClaw 是个"吃 Token 大户"——每次对话都要带上完整的系统提示、工具定义、记忆文件等上下文，单次请求动辄消耗上万 Token。如果选错了模型或套餐，要么经常被限流搞得体验稀烂，要么月底账单让人心梗。

这篇文章对比国内五家主流 AI 厂商的 Coding Plan / API 定价和限流策略，聚焦一个问题：哪个最适合搭配 OpenClaw 使用？

本文数据截至 2026 年 3 月，价格和限流策略可能随时调整，请以官方最新信息为准。

一、各家 Coding Plan / API 概览

1.1 智谱 GLM Coding Plan

智谱推出了专门的 Coding Plan 套餐，面向 Cursor、Copilot、OpenClaw 等 AI 编码工具用户。

套餐月费定位建议并发项目数
Lite¥49/月（包年¥34/月）个人轻量~80 prompts/5h
Pro（最受欢迎）¥149/月（包年¥104/月）个人主力~400 prompts/5h
Max¥469/月（包年¥328/月）重度用户~1,600 prompts/5h

优势：

  • Lite 套餐仅支持 GLM-4.7/4.6（GLM-5 需 Pro 及以上）；Pro/Max 才支持 GLM-5（744B MoE），且 GLM-5 消耗 3 倍额度
  • 套餐制，不用操心 Token 计费
  • 低峰期动态提升并发

劣势：

  • 按 prompt 配额计费而非无限用——有开发者反馈 Pro 套餐在工作日高峰期（下午 15:00-18:00）连基础请求都频繁被拦截
  • 套餐用户不支持申请调整并发，固定等级
  • 曾因限流问题在 V2EX 社区引发讨论（来源：前沿哨所报道）

OpenClaw 适配度： 适合单人、单渠道、低频场景。如果同时挂多个 channel（飞书+Discord+WhatsApp），并发可能不够用。

1.2 DeepSeek API

DeepSeek 不搞套餐制，纯按 Token 计费，简单粗暴。当前主力模型为 DeepSeek V3.2（685B MoE）。

模型输入价格（¥/百万Token）输出价格（¥/百万Token）上下文窗口

优势：

  • 价格极低，缓存命中后输入仅需 ¥0.19/百万 Token
  • OpenAI 兼容 API，OpenClaw 零成本接入
  • 无套餐限制，按量付费，不用的不收钱

劣势：

  • 纯文本，无多模态
  • 高峰期偶有延迟波动
  • 需要自己关注用量，没有"无限用"的安全感

1.3 阿里云百炼（通义千问 Qwen）

通过阿里云百炼平台调用 Qwen 系列，按 Token 计费，免费额度非常大方。

限流规则： 基于 API Key 级别限流，触发限流后通常 1 分钟内恢复。支持 Batch API（批量推理），不受实时限流约束。

优势：

  • 限流极其宽松（30,000 RPM），适合高并发场景
  • 免费额度大，够用几个月
  • 模型矩阵丰富，可按任务路由不同模型
  • 阿里云基础设施稳定，国内延迟低

三、我的推荐

🏆 最佳性价比：阿里云百炼 + Qwen 3.5 Plus/Flash

理由：

  1. 限流极其宽松 （30,000 RPM），多渠道并发完全不是问题
  2. 新用户免费 7000 万 Token ，够用几个月零成本体验
  3. 模型矩阵丰富 ，可以给 OpenClaw 配置模型路由——简单任务走 Flash（¥0.8/百万Token），复杂任务走 Plus（¥2/百万Token）
  4. 阿里云基础设施 稳定可靠，国内延迟低
  5. OpenAI 兼容 API，OpenClaw 直接配置即可

💰 最省 Token 方案：MiniMax Token Plan

理由：

  1. OpenClaw 的系统提示每次都很长（SOUL.md + 工具定义 + 记忆文件），按 Token 计费很吃亏
  2. MiniMax Token Plan 按请求次数 计费，一次对话不管消耗多少 Token 都算 1 次
  3. Plus 套餐 ¥49/月，1,500 次/5小时，日常使用绑绑有余
  4. 额外赠送语音、图像等多模态能力

🆓 最省钱过渡：DeepSeek + 免费额度

理由：

  1. 注册送 500 万 Token，够轻度使用
  2. 缓存命中后价格极低（¥0.19/百万 Token）
  3. OpenClaw 的系统提示每次都一样，缓存命中率会很高

适合人群： 个人轻度使用，预算敏感。

🎁 免费白嫖：NVIDIA NIM

完全免费，零 Token 费。但 40 RPM 太低，多渠道场景会频繁触发限流。适合只挂一个渠道的个人用户。

⚠️ 不推荐：智谱 Coding Plan（当前阶段）

按 prompt 配额计费（~400/5小时），OpenClaw 单次对话消耗约 1-3 个 prompt，配额消耗极快；高峰限流也有反馈。Max 套餐 ¥469/月性价比不如 MiniMax Max（¥119/月，4,500 次/5小时）。 MiniMax Max ¥119/月（4,500 次/5小时）或百炼 Qwen Plus 按量计费都更划算。建议保留智谱 Lite（免费）作为备用。

四、配置建议

以多供应商容灾为目标，推荐 OpenClaw 的模型配置：

主模型（日常对话）：qwen3.5-plus
轻量模型（心跳/简单任务）：qwen3.5-flash
备用模型（容灾）：deepseek-chat
尝鲜/多模态：minimax-m2.7

多供应商容灾原则： 不要把所有鸡蛋放在一个篮子里。主用百炼，备用 DeepSeek 或 MiniMax，任何一家挂了自动切到另一家。

五、写在最后

AI 模型的价格战还在继续，今天的结论可能下个月就变了。但有几个原则不会变：

  1. 限流比价格更重要 ——便宜但不能用，等于没用
  2. 计费模式要匹配使用方式 ——OpenClaw 长上下文场景适合按请求计费（MiniMax）或限流宽松的按量计费（百炼）
  3. 模型路由比单一模型更聪明 ——不同任务用不同模型
  4. 缓存是省钱利器 ——选支持上下文缓存的供应商
  5. 永远要有备用方案 ——多供应商配置是必须的
