> 旁路由模式下跑 OpenClash 或 PassWall，代理节点"时通时断"，直连却一切正常？这篇文章帮你定位根因并彻底解决。

## 📐 典型网络架构 
    
    
    光猫 → 爱快路由器(拨号) → 旁路由(OpenWrt) → AC(交换机) → AP → 终端设备
              .168.1.2            192.168.1.100
    

## 🔍 问题现象 

  * 所有节点都会出现"时通时断"现象
  * 手机/电脑直连节点（非路由器代理）正常，说明机场节点本身稳定
  * OpenClash 和 PassWall 两个插件都有同样问题



## 🎯 根因分析 

旁路由模式下，**网关和 DNS 必须同时指向旁路由** 。

### 错误配置示例 
    
    
    网关：192.168.1.100（旁路由）✅
    DNS：192.168.1.2（爱快）或运营商 DNS ❌
    

### 问题后果 

  * TCP 流量走旁路由代理 → 部分连接正常
  * DNS 查询绕过旁路由 → 域名解析失败或超时
  * 表现为：网站"一会儿能用一会儿超时"



## ⚡ 快速验证方法 

在需要代理的终端设备上手动设置网络：
    
    
    网关：192.168.1.100
    DNS：192.168.1.100（关键！）
    

设置后测试国内外网站，如果稳定则确认是 DNS 配置问题。

## 🛠️ 解决方案 

### 方案一：爱快 DHCP 全局代理（推荐） 

适用场景：希望所有设备都走代理

爱快后台 → **DHCP 服务** ：
    
    
    网关地址：192.168.1.100
    DNS 服务器：192.168.1.100
    

### 方案二：爱快 DHCP 默认，设备手动配置 

适用场景：只需部分设备代理

爱快 DHCP 保持默认（网关 192.168.1.2），在需要代理的设备上：

**Windows** ：
    
    
    网络连接 → IPv4 属性
    网关：192.168.1.100
    DNS：192.168.1.100
    

**macOS / 手机** ：
    
    
    WiFi 设置 → DNS 手动配置 → 192.168.1.100
    

## ⚙️ OpenClash 配置要点 

旁路由模式下的推荐设置：

### 1\. 开始页面 

  * 模式：**规则模式** 或**黑名单模式**
  * 启用：勾选"开启"



### 2\. DNS 设置 

  * **DNS 劫持：开启**
  * 劫持方式：**Fake-IP-Enhanced** 或 **重定向**
  * Fallback DNS：`223.5.5.5` 或 `119.29.29.29`



### 3\. 重定向 

  * 开启：勾选



### 4\. TUN 模式（可选，更稳定） 

  * 开启：勾选



## ⚠️ 常见误区 

误区 | 正确做法  
---|---  
只改网关，不改 DNS | **网关和 DNS 都指向旁路由**  
两个 DHCP 都开着 | **只能有一个 DHCP 服务器**  
不检查防火墙规则 | 爱快防火墙可能拦截旁路由流量  
OpenClash 关闭 DNS 劫持 | 旁路由模式必须开启 DNS 劫持  
  
## 🔧 额外检查项 

如果上述配置后仍不稳定：

  1. **爱快防火墙** ：检查安全设置、端口映射，确保 7890/7892/7893 端口未被拦截
  2. **爱快行为管理** ：检查访问控制策略
  3. **旁路由资源占用** ：`top` 命令查看 CPU/内存使用情况
  4. **OpenWrt 防火墙** ：检查端口转发规则



## 💡 总结 

旁路由不稳定问题 **90% 是 DNS 配置错误** 导致。确保终端设备的 DNS 指向旁路由，问题通常就能解决。
