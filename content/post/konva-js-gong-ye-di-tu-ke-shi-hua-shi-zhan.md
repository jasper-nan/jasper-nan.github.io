在工业自动化领域，一张好用的地图监控大屏直接影响运维效率。本文记录我在**远舢 AIO（All-in-One）工业自动化管理平台** 中，使用 **Konva.js** 实现 AGV 车队地图可视化的完整实践——包括坐标系翻转、贝塞尔曲线拐角、SSE 实时推送等核心方案。

## 项目背景

远舢 AIO 是一个基于 Django 的工业自动化管理系统，核心模块包括：

  * **TCS** ：交通控制系统，负责 AGV 车队调度
  * **MCS Mock** ：设备模拟器（电梯、穿梭车、输送线）
  * **Outdoor AGV Mock** ：室外无人车仿真，含 Konva.js 地图可视化大屏



地图数据来自真实厂区，包含 469+ 个路径节点、充电桩、停车点、业务区（高架库/醇化库）和限制区域。核心挑战是：**如何用 Canvas 2D 高效渲染复杂工业地图，并支持 50+ 车辆的实时位置更新？**

## 为什么选 Konva.js

在选型阶段对比了几种方案：

方案| 优点| 缺点  
---|---|---  
Leaflet/Mapbox| 地理地图成熟| 不适合厂区平面布局，坐标系难以自定义  
原生 Canvas API| 无依赖| 交互事件、图层管理全靠自己写，工作量巨大  
**Konva.js**|  图层管理完善、事件系统成熟、性能优秀| 需自行处理坐标转换  
  
最终选择 **Konva.js** （v8.x），因为它在图层隔离、事件委托、动画支持上做得非常到位，且对工业场景的自定义坐标系友好。

## 核心架构：多图层分离

不要把所有元素画在一个图层上。我们采用**四图层分离架构** ：
    
    
    this.mapLayer     = new Konva.Layer({ name: 'map' });       // 地图连线、节点、区域
    this.labelLayer   = new Konva.Layer({ name: 'labels' });    // 文字/图标（不随地图旋转）
    this.vehicleLayer = new Konva.Layer({ name: 'vehicles' });  // 车辆、轨迹
    this.alertLayer   = new Konva.Layer({ name: 'alerts' });    // 报警提示

分层的好处：

  * 地图层和车辆层**独立刷新** ，车辆移动时不重绘地图
  * 文字标签单独一层，方便做**不随缩放旋转** 的处理
  * 报警层永远在最上方，不受其他元素遮挡



## 踩坑一：Y 轴翻转

工业地图的坐标系是**数学坐标系** （Y 轴向上），而 Canvas 是**屏幕坐标系** （Y 轴向下）。如果不处理，地图会上下颠倒。

最直接的方案：对地图图层做 Y 轴翻转：
    
    
    this.mapLayer.scaleY(-1);

但这会导致文字也颠倒。解决方式：所有 `Konva.Text` 单独设置 `scaleY: -1` 抵消图层翻转，保证标签始终水平可读。

## 踩坑二：贝塞尔曲线拐角

厂区的 AGV 路径不是简单的折线，在拐弯处需要**平滑圆弧** 过渡。我们实现了自动贝塞尔曲线检测：

  * 遍历路径节点，计算相邻线段的夹角
  * 当夹角在 **20°~170°** 之间时，判定为拐角
  * 用 `Konva.Path` 绘制二次贝塞尔曲线连接
  * 支持 `trim` 裁剪，避免曲线侵入节点区域


    
    
    // 二次贝塞尔路径示例
    const path = new Konva.Path({
        data: `M ${x1} ${y1} Q ${cx} ${cy} ${x2} ${y2}`,
        stroke: '#666',
        strokeWidth: 2
    });

效果：469 个节点的完整地图渲染后，拐角自然流畅，AGV 行驶轨迹更符合真实物理运动。

## 实时车辆渲染与 SSE

车辆状态通过后端 **SSE（Server-Sent Events）** 实时推送，数据格式：
    
    
    {
      "id": "AGV-001",
      "x": 1250.5,
      "y": 3680.0,
      "heading": 90,
      "status": "moving",
      "battery": 87
    }

前端 `updateVehicle()` 接收数据后，更新 `vehicleLayer` 中的车辆 Group：
    
    
    // 车辆是一个 Group：车身矩形 + 车头箭头 + 前后轴小圆点 + ID/电量标签
    const vehicleGroup = new Konva.Group({
        x: canvasX, y: canvasY, rotation: heading
    });
    vehicleGroup.add(new Konva.Rect({ width: 40, height: 20, fill: '#1890ff' }));
    vehicleGroup.add(new Konva.Arrow({ points: [0, 0, 20, 0], fill: '#fff' }));
    // ...

轨迹用 `Konva.Line` 保存最近 **500 个点** ，超出自动丢弃，避免内存泄漏。

## 交互设计

除了实时渲染，还实现了完整的交互能力：

  * **滚轮缩放** ：以鼠标位置为中心点缩放
  * **中键拖拽平移** ：查看大地图的不同区域
  * **点击车辆弹窗** ：显示详细状态（任务、电量、速度）
  * **Hover 高亮** ：节点、业务区、限制区的鼠标悬停反馈

## 源码与总结

远舢 AIO 的完整代码暂未开源，但 Konva.js 的地图可视化核心逻辑已整理成独立模块。关键经验：

  1. **图层分离** 是性能的基础，不要所有元素堆在一起
  2. **Y 轴翻转** 要注意文字的正立处理
  3. **贝塞尔曲线** 让工业路径更专业，但角度检测阈值需根据实际路径调整
  4. **SSE 比 WebSocket 更轻量** ，适合单向高频推送场景
  5. Konva.js 的 `Group` 组合能力极强，车辆、充电桩等复杂元素建议封装为 Group
